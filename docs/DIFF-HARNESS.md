# Differential Harness

Focused differential coverage lives in [`test/differential.test.ts`].

- Oracle: OpenSSL CLI (`openssl`) with normalized verdict parsing, not raw text
  snapshots.
- Goal: catch interop regressions across path validation, direct CRL revocation,
  issuer-signed OCSP, and DNS/IP identity matching.
- Run:

  ```bash
  bun test test/differential.test.ts
  ```

## Covered comparisons

| Area            | micro509 entrypoint                                 | OpenSSL command family                                         | Compared output                                                      |
| --------------- | --------------------------------------------------- | -------------------------------------------------------------- | -------------------------------------------------------------------- |
| Path validation | `verifyCertificateChain()`                          | `openssl verify`                                               | pass/fail verdict, plus coarse failure class for invalid chains      |
| Direct CRL      | `checkCertificateRevocationAgainstCrl()`            | `openssl verify -crl_check` + `openssl crl -crlnumber -issuer` | good/revoked verdict, issuer, CRL number                             |
| OCSP            | `validateOcspResponse()` + `parseOcspResponseDer()` | `openssl ocsp` responder/client mode                           | response accepted/rejected and leaf cert status (`good` / `revoked`) |
| Identity        | `matchServiceIdentity()`                            | `openssl x509 -checkhost` / `-checkip`                         | match/mismatch for DNS and IP                                        |

## Intentional scope limits

- Compare normalized verdicts/statuses only.\
  OpenSSL CLI diagnostics are useful, but not stable enough to diff
  line-for-line.
- Keep OCSP coverage to issuer-signed responses generated by OpenSSL responder
  mode.\
  Delegated responder, nonce, and local-policy edge cases stay covered by the
  focused OCSP tests.
- Keep CRL differential coverage to direct complete CRLs.\
  Delta and indirect CRL semantics stay covered by the dedicated CRL/revocation
  suites.
- Keep identity differential coverage to DNS and IP.\
  URI-ID, SRV-ID, IDNA edge cases, and CN-fallback semantics stay covered by the
  dedicated identity tests.
- Keep policy comparison at verdict level only.\
  `micro509` exposes richer typed policy outcomes than OpenSSL's CLI.

## Known oracle differences to document, not overfit

- `micro509` returns typed failure codes/details;
  OpenSSL mostly returns exit codes plus human-readable diagnostics.
- DNS CN fallback is a compatibility mode in `micro509`,
  not a default comparison path to copy from OpenSSL behavior.
- OpenSSL's CLI is a good mature interoperability oracle,
  but not a full structural parse oracle for every RFC surface this repo exposes.
