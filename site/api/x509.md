### buildCertificateExtensions()

> **buildCertificateExtensions**(`subjectPublicKeyInfo`, `issuerPublicKeyInfo`, `input`, `subjectIsEmpty?`): `Uint8Array`\<`ArrayBufferLike`\>[]

Defined in: [x509/extensions.ts:683](https://github.com/kjanat/ts-x509/blob/vitepress/src/x509/extensions.ts#L683)

Build the v3 extensions block for a certificate.

Automatically adds SKI, AKI (when issuer key is available), and
basicConstraints (defaults to `{ ca: false }`). Additional extensions
come from the caller's [`CertificateExtensionsInput`](micro509.md#certificateextensionsinput).

#### Parameters

##### subjectPublicKeyInfo

`Uint8Array`

DER-encoded SPKI of the subject.

##### issuerPublicKeyInfo

DER-encoded SPKI of the issuer, or `undefined` for self-signed.

`Uint8Array`\<`ArrayBufferLike`\> | `undefined`

##### input

Optional extension configuration.

[`CertificateExtensionsInput`](micro509.md#certificateextensionsinput) | `undefined`

##### subjectIsEmpty?

`boolean` = `false`

Whether the certificate subject DN is empty. When `true`, the
subjectAltName extension is marked critical per RFC 5280 §4.2.1.6.

#### Returns

`Uint8Array`\<`ArrayBufferLike`\>[]

Array of DER-encoded Extension SEQUENCEs.

### encodeName()

> **encodeName**(`input`): `Uint8Array`

Defined in: [x509/name.ts:196](https://github.com/kjanat/ts-x509/blob/vitepress/src/x509/name.ts#L196)

DER-encodes an X.509 `Name`.

Returns a DER `SEQUENCE` of RelativeDistinguishedNames (RDNs).\
Each RDN emitted by this helper contains exactly one name attribute.

#### Parameters

##### input

[`NameInput`](micro509.md#nameinput)

Name fields in convenience-object form or caller-ordered attribute form.

#### Returns

`Uint8Array`

DER-encoded X.509 `Name` bytes.

#### See

[RFC 5280 Appendix A.1](https://datatracker.ietf.org/doc/html/rfc5280#appendix-A.1)

[`NameObject`](micro509.md#nameobject) input emits populated fields in the canonical order from `NAME_OBJECT_ORDER`.\
[`NameAttribute`](micro509.md#nameattribute) array input preserves caller-supplied ordering,
but each entry still becomes its own single-attribute RDN.

Attribute OIDs and ASN.1 string encodings come from `NAME_FIELD_DEFINITIONS`.\
Empty strings and `undefined` fields are ignored when the input is a [`NameObject`](micro509.md#nameobject).

#### Examples

```ts
const der = encodeName({
  country: 'US',
  commonName: 'example.com',
});

// emits two single-attribute RDNs: C=US, then CN=example.com
```

```ts
const der = encodeName([
  { type: 'country', value: 'US' },
  { type: 'commonName', value: 'example.com' },
]);

// preserves caller order: C first, then CN
```

#### Throws

If the input produces no attributes, contains an unsupported field key, or uses an invalid country code.

### encodeRelativeDistinguishedName()

> **encodeRelativeDistinguishedName**(`attributes`): `Uint8Array`

Defined in: [x509/name.ts:236](https://github.com/kjanat/ts-x509/blob/vitepress/src/x509/name.ts#L236)

DER-encodes a single RelativeDistinguishedName (RDN).

Returns a DER `SET OF` name attributes for one X.509 name segment.\
Use this when you need a multi-valued RDN.

#### Parameters

##### attributes

[`RelativeDistinguishedNameInput`](micro509.md#relativedistinguishednameinput)

Attribute list to encode inside one RDN.

#### Returns

`Uint8Array`

DER-encoded RelativeDistinguishedName bytes.

#### See

[RFC 5280 Appendix A.1](https://datatracker.ietf.org/doc/html/rfc5280#appendix-A.1)

Attribute OIDs and ASN.1 string encodings come from
`NAME_FIELD_DEFINITIONS`.

#### Example

```ts
const rdn = encodeRelativeDistinguishedName([
  { type: 'commonName', value: 'example.com' },
  { type: 'serialNumber', value: 'device-7' },
]);

// emits one RDN with both attributes in the same SET
```

#### Throws

If the attribute list is empty, contains an unsupported field key, or uses an invalid country code.
