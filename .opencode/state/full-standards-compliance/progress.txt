## Codebase Patterns

- ESM-only, WebCrypto-only, platform-neutral core; avoid Node builtins in core paths.
- TypeScript rules are strict: no `any`, no non-null assertions, no `as Type`, no classes.
- Tabs, single quotes, `.ts` import extensions, readonly-heavy interfaces, discriminated unions.
- Keep path validation, service identity, and revocation as separate public concepts.
- Package exports are driven by `tsdown.config.ts`; do not hand-edit generated `package.json` exports.
- For stable package subpaths, prefer a named tsdown `entry` object; let `exports: true` derive package exports from entry keys.
- Root `micro509` export should stay ergonomic; granular domains should live behind stable subpath exports.
- Distribution-point model change is a clean break; no temporary URI-only compat alias.
- RFC 5280 `directoryName` comparison is semantic: ordered DN, unordered RDN sets, RFC 4518-prepared AVA strings; never compare raw DER prefixes.
- RFC 9618 policy validation should use a depth-indexed DAG keyed by `(depth, validPolicy)` plus constrained policy sets; never build or expose the deprecated RFC 5280 policy tree during core validation.
